## Summary

Address UI and admin findings: use PrivilegeLevel enum in admin layout; normalize term dates in TermsManagement and remove redundant cast; fix CommitteeSelector occupant name (no dangling comma, VRCNUM fallback); handle getActiveTermId() errors and use CommitteeMembershipStatus.SUBMITTED on requests page.

## Acceptance Criteria

- [ ] In admin layout: use `privilegeLevel={PrivilegeLevel.Admin}` and import `PrivilegeLevel` from `@prisma/client` (replace string literal "Admin").
- [ ] In TermsManagement: ensure parseTermList returns Date for startDate/endDate and use `format(term.startDate, "MMM d, yyyy")` without extra `new Date()` or cast; or use a small normalizer helper.
- [ ] In CommitteeSelector: build occupant display name from non-empty parts (e.g. `[lastName, firstName].filter(Boolean).join(", ")`); fall back to `occupant.voterRecord.VRCNUM` only when both name parts are empty.
- [ ] In committees/requests page: wrap getActiveTermId() and findMany in try/catch; on error render empty state or "No active term" message. Use `status: CommitteeMembershipStatus.SUBMITTED` and import `CommitteeMembershipStatus` from `@prisma/client`.
