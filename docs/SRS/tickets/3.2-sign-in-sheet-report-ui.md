**Status:** Open
**Roadmap:** [SRS_IMPLEMENTATION_ROADMAP.md](../SRS_IMPLEMENTATION_ROADMAP.md) §3.2

**Depends on:** [3.0 Report-server Migration](../SRS_IMPLEMENTATION_ROADMAP.md), [3.0a Report Audit](../SRS_IMPLEMENTATION_ROADMAP.md)

## Summary

Add sign-in-sheet report generation to frontend reports UX with clear parameter defaults and role-scoped behavior.

## Acceptance Criteria

- [ ] Add `Sign-in Sheet` to report picker with user-friendly naming.
- [ ] Define and implement required parameters (committee/jurisdiction/date, as finalized).
- [ ] Leader view is jurisdiction-scoped by default; admin can run countywide.
- [ ] Parameter validation and empty states are defined (no committees, missing date, etc.).
- [ ] Output triggers existing report pipeline without introducing report-specific UI forks.
