# 3.7 LTED Crosswalk Import UI/Admin Flow

**Status:** Open
**Roadmap:** [SRS_IMPLEMENTATION_ROADMAP.md](../SRS_IMPLEMENTATION_ROADMAP.md) Tier 3 follow-up
**Effort:** 0.5–1 day
**Depends on:** [1.1b LTED-to-Assembly-District Mapping](1.1b-lted-assembly-district-mapping.md), [3.1 Jurisdiction Assignment UI](3.1-jurisdiction-assignment-ui.md)

## Summary

Define and implement the admin UX for crosswalk import/update with a clear entry point, format handling, validation, and operator feedback.

## Acceptance Criteria

- [ ] Select final v1 input mode(s): fixed CSV, mapped CSV, manual fallback (documented decision).
- [ ] Define admin entry point and placement in IA/navigation.
- [ ] Implement upload/entry flow with validation feedback and row-level error reporting.
- [ ] Show deterministic import summary (`rows processed`, `created`, `updated`, `failed`).
- [ ] Document update cadence/operator guidance for ongoing maintenance.
