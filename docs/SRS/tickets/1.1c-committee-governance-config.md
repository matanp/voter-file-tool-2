# 1.1c Committee Governance Config

**Status:** Done
**Roadmap:** [SRS_IMPLEMENTATION_ROADMAP.md](../SRS_IMPLEMENTATION_ROADMAP.md) §1.1c
**Effort:** 0.5–1 day
**Depends on:** None

## Summary

Create the `CommitteeGovernanceConfig` model and `IneligibilityReason` enum to externalize eligibility rules (party, capacity, AD check) and support admin override configuration. Replace hardcoded values with config-driven checks.

## Acceptance Criteria

- [x] New `CommitteeGovernanceConfig` model: `requiredPartyCode`, `maxSeatsPerLted`, `requireAssemblyDistrictMatch`, `nonOverridableIneligibilityReasons: IneligibilityReason[]` (seed default `[]`)
- [x] Add `IneligibilityReason` enum (NOT_REGISTERED, PARTY_MISMATCH, ASSEMBLY_DISTRICT_MISMATCH, CAPACITY, ALREADY_IN_ANOTHER_COMMITTEE)
- [x] Migration + seed row for MCDC defaults (`requiredPartyCode="DEM"`, `maxSeatsPerLted=4`, `requireAssemblyDistrictMatch=true`)
- [x] Config consumed by seat/capacity implementation paths in Phase 1; 2.1 eligibility service is explicitly tracked as Phase 2 follow-on

## Implementation Notes

- **Schema:** See [SRS_DATA_MODEL_CHANGES.md](../SRS_DATA_MODEL_CHANGES.md) — CommitteeGovernanceConfig, IneligibilityReason enum
- **Singleton:** Single row per deployment for v1; future may add org/tenant FK for multi-tenant
- **Admin override:** `nonOverridableIneligibilityReasons` is a blacklist; empty = all rules overridable. Add IneligibilityReason values to tighten policy over time. See [SRS_IMPLEMENTATION_ADMIN_OVERRIDE.md](../SRS_IMPLEMENTATION_ADMIN_OVERRIDE.md)
- **No admin UI in v1:** Config via seed/migration only; see Future Considerations in roadmap
